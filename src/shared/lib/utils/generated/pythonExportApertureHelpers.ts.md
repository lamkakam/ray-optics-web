# `shared/lib/utils/generated/pythonExportApertureHelpers.ts`

## Purpose

Generated TypeScript module that exports the standalone Python aperture helper source used by `buildExportScript`.

## Exports

```ts
export const pythonExportApertureHelpers: string;
```

## Behavior

- The file is generated by `scripts/generate-python-export-helpers.mjs` and should not be edited by hand.
- The generated `.ts` file is ignored by git. Normal npm lifecycle hooks regenerate it before install/check/test/build commands that consume TypeScript.
- The exported string is the concatenation of:
  1. `src/python/src/rayoptics_web_utils/aperture/annular.py`
  2. `src/python/src/rayoptics_web_utils/aperture/offset_circular.py`
  3. `src/python/src/rayoptics_web_utils/aperture/offset_rotated_rectangular.py`
- Source files are joined with a single newline.
- The generator writes the string with `JSON.stringify`, so Python syntax, docstrings, and backslashes cannot break TypeScript parsing.

## Usages

- Imported by `src/shared/lib/utils/pythonScript.ts` and interpolated into the export notebook preamble before model construction.
- Imported by `src/shared/lib/utils/__tests__/pythonScript.test.ts`, after `pretest` regenerates the file, to assert exported notebook scripts include the Python-source helper block.
